# Table-Based Output Style

表格样式是最适合结构化数据展示的输出格式，能够清晰地组织信息，便于比较和分析。

## 核心特性

- ✅ **清晰的列结构**：信息按列组织，易于阅读
- ✅ **对齐格式**：数值和文本正确对齐
- ✅ **表格边框**：明确的视觉分隔
- ✅ **标题行**：突出显示列标题
- ✅ **斑马纹行**：提高可读性
- ✅ **支持复杂数据**：适合多层嵌套信息

## 使用方法

激活表格输出样式：

```
/output-style table-based
```

然后提供需要表格化展示的信息。

## 表格结构

```
| 列标题1 | 列标题2 | 列标题3 |
|---------|---------|---------|
| 数据1   | 数据2   | 数据3   |
| 数据4   | 数据5   | 数据6   |
```


### 2. 复杂表格

```

```

## 表格格式化技巧

### 1. 对齐方式
- **左对齐**：文本内容（默认）
- **右对齐**：数值和金额
- **居中**：标题和重要信息

### 2. 数据类型
- **文本**：保持原始格式
- **数字**：自动格式化千分位
- **百分比**：显示%符号
- **布尔值**：✅ 或 ❌

### 3. 特殊标记
- **状态**：🔴进行中 ✅完成 ⏳待开始
- **优先级**：🔥紧急 ⚡重要 📋普通
- **进度**：数字+百分比

## 最佳实践

### 1. 表格设计
- 保持列数适中（建议不超过6列）
- 使用有意义的列标题
- 确保数据对齐一致
- 突出重要的行或列

### 2. 数据内容
- 提供完整的比较维度
- 使用一致的单位和格式
- 包含必要的说明和注释
- 标注数据来源和更新时间

### 3. 可读性优化
- 使用颜色标记不同状态
- 合理的行高和列宽
- 分组相关信息
- 提供摘要和结论

## 局限性

- 复杂的多层数据可能难以展示
- 大量行数据可能超出显示范围
- 移动设备上可能需要横向滚动
- 不适合非结构化的自由文本
